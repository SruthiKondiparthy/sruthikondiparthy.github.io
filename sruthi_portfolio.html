MyGermanFreund
"Welcome to Germany — We've Got You Covered"

A Flutter mobile app that guides expats, international students, and immigrants through the chaos of settling in Germany — Anmeldung, housing, health insurance, jobs — with AI-powered checklists and a letter scanner that translates German documents instantly.

Product Lifecycle
Discovery Define Design Build Launch Scale
Flutter/Dart Firebase Hosting Google ML Kit OCR GitHub Actions CI/CD ✦ On-device OCR (offline) ✦ Translation Pipeline ✦ DocumentAnalyzer
💡 The Idea
Discovered the pain firsthand as an immigrant in Germany: government processes are opaque, bureaucracy is in German only, and newcomers waste weeks on tasks that should take hours. Validated with user interviews across 3 target groups — expats, international students, and migrant families — identifying 10+ high-friction pain points around Anmeldung, bank accounts, visas, housing, and healthcare.
🗺 PM Journey Across the Lifecycle
Discovery: Conducted user interviews & contextual analysis; mapped Customer Journeys (emotions, tasks, needs, wants) across 10 newcomer scenarios. Built HMW statements and generated 40+ ideas through structured ideation sprints.

Define: Defined Product Vision Canvas; formulated Vision Statement; set OKRs targeting 50k active users in Year 1 with Daily Active Users as North Star KPI. Completed Business Model Canvas with freemium + subscription + strategic partnerships model.

Design: Created User Flows, Low-Fi Wireframes, and Hi-Fi Figma prototypes (mobile-first). Mapped 5-step product backlog with Epics → User Stories in Jira. Wrote acceptance criteria using scenario-based & checklist formats.

Build: Owned discovery-to-delivery for Flutter/Firebase marketplace. Authored versioned API contracts, integration architecture, and error standards. Implemented event instrumentation, idempotency, retries, and reconciliation playbooks. Set up Agile Scrum with Sprint Planning, Retrospectives, and SMART sprint goals.

Validate & Measure: Designed Assumptions Mapping (desirability/feasibility/viability). Created One-Page Test Plan and Bug Matrix. Ran Landing Page Demand Test (Fake Door) to validate core value proposition. Defined KPIs using 6-A KPI Canvas; built CI/CD pipeline with canary rollout strategy.
🤖 How I Used AI
Designed and shipped an on-device document intelligence pipeline: Google ML Kit OCR extracts text from German official letters without a server round-trip, a custom DocumentAnalyzer then parses dates, deadlines, IBAN, amounts, and contact details — turning an intimidating letter into a structured action list in seconds. Built AI into the development workflow too: Copilot for OCR and async implementation, LLM consultation for UX fallback design and stack decisions.
📊 Market & Impact
TAM: ~10M newcomers in Germany. SAM: ~2M digitally active expats. SOM: 50k users target Year 1. Competitor analysis across 12 platforms — MyGermanFreund is uniquely positioned as the only "Easy + Comprehensive" solution combining bureaucracy, housing, jobs, healthcare, and social integration in one app.

Applify AI
"CV + Cover Letter, job-tailored, German-standard — one click"

In Germany, an Anschreiben (cover letter) is not optional — it is often the deciding factor. Applify takes a job description and your background, then generates both a German-formatted Lebenslauf and a tailored Anschreiben in one click, following Bundesagentur für Arbeit standards. Fully offline. Exports PDF and DOCX instantly.

Product Lifecycle
Discovery Define Build Launch Scale
Python 3.12 · FastAPI Next.js 15 · React 18 · TypeScript TailwindCSS · Zustand Jinja2 · python-docx · ReportLab Docker · GitHub Actions · Vercel ✦ DeepSeek / Ollama (offline LLM) ✦ PDF & DOCX generation
💡 The Idea
German applications require both a Lebenslauf and an Anschreiben — and both have strict, unforgiving conventions. A generic cover letter or a missing photo is as damaging as wrong formatting. Skilled immigrants face this double barrier daily. Applify removes it: paste a job description, add your background, get a job-tailored CV and cover letter pair — formatted to Bundesagentur standards, export-ready in one click.
🗺 PM Journey Across the Lifecycle
Discovery: Core insight — skilled immigrants are not rejected for lack of ability, but for unknown rules, unfamiliar processes, and poor application positioning. Desk research on German CV standards and common rejection reasons for non-native applicants confirmed the top failure points: wrong format, missing photo, wrong section order, generic Anschreiben, and no ATS keyword alignment.

Define: Scoped MVP tightly — one core flow: input → AI analysis → German-format output. Defined success metrics: CV completion rate, download rate, and user return rate. Mapped user stories and acceptance criteria for 3 key user stories (AI CV Generation, Anschreiben Generation, Job Description Matching).

Build: FastAPI REST backend (/generate-resume) with versioning, structured JSON schemas, and robust error handling. Integrated Ollama/DeepSeek for fully offline LLM inference. Enforced German Lebenslauf + Anschreiben formatting via Jinja2 templates based on Arbeitsagentur rules. Built PDF (ReportLab) + DOCX (python-docx) dual-export pipeline. Next.js 15 frontend (TypeScript, TailwindCSS, Zustand) deployed on Vercel. Docker containerised, GitHub Actions CI/CD.

Validate: Used structured prompt templates and verification checklists to keep AI-generated output traceable and reproducible. Each output validated against German DIN formatting checklist before export.

Roadmap: LinkedIn import · ATS compatibility checker · Europass CV export · multi-language CV support · German interview preparation (technical & non-technical) with role-specific Q&A practice.
🤖 How I Used AI
Core product is AI — LLM (DeepSeek via Ollama) rewrites CV content to match job description keywords, enforces German formatting rules, and generates tailored cover letters. Used structured output validation with JSON schema enforcement to prevent hallucinations. Built reusable prompt templates with role/context/constraint pattern. AI also used during development: architecture validation, code review prompts, edge case generation, and documentation drafting.
⚙ Technical Highlights
Backend: FastAPI (Python 3.12) REST endpoint returning structured JSON — cv_text, cover_letter_text, easy-language variants, PDF and DOCX as base64. Jinja2 templates enforce Bundesagentur formatting rules. Local LLM inference via Ollama/DeepSeek — no OpenAI billing, fully offline. PDF via ReportLab, DOCX via python-docx.

Frontend: Next.js 15 (React 18, TypeScript), TailwindCSS, Zustand for state management, Axios for API calls — deployed on Vercel.

Infrastructure: Docker-ready, GitHub Actions CI/CD, environment secrets isolation via .env.
